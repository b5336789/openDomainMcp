使用說明書 · User Guide

把文件與程式碼，
收進一座能對話的知識庫

openDomainMcp 會讀進你交付的檔案，交由 Claude 整理出其中的領域知識，轉成向量後存放起來。 日後想找某段內容時，用平常說話的方式描述即可，散落在各份文件與原始碼裡的答案都能一併找回。

概觀

一套通用的領域知識工作流

無論是 PDF、Word、Markdown，或十多種程式語言的原始碼，放進來即可。 程式碼會依語法結構切分，文件則按段落遞迴切分；處理完之後，命令列、MCP 與網頁三種介面查的都是同一份資料。

🗂️

什麼都收得進來

常見格式 txt、md、pdf、docx、html、json、csv 一應俱全；其餘檔案只要是 UTF-8 純文字，同樣照收。

🌳

依語法結構切分程式碼

切塊的邊界落在函式、類別與方法上，而不是按固定字數硬切。這個作法借鏡自開源專案 claude-context。

🧠

連同重點一起入庫

每一段內容都先由 Claude 寫下摘要與關鍵概念，再與原文一同嵌入，因此搜尋比對的是語意，而非字面文字。

Quick Start

三分鐘上手

環境只需要 Python 3.11 以上。第一次執行時，本地的嵌入模型會自動下載一個輕量模型，往後便不再需要連線。

  1. 安裝　建立虛擬環境、安裝套件，再把設定範本複製一份留用。 
    # 建立並啟用虛擬環境
python -m venv .venv && source .venv/bin/activate
# 安裝（含開發相依）
pip install -e ".[dev]"
cp .env.example .env   # 依需求調整
  2. 匯入資料　指向單一檔案或整個資料夾都可以，目錄會自動遞迴掃描。 
    opendomainmcp ingest ./your-project
# [load] … [split] … [extract] … [store]
# Indexed 42 files / 318 chunks.
  3. 開始查詢　用一句話描述你要找的東西即可；需要時加上 --kind，限定只找程式碼或只找文件。 
    opendomainmcp search "檢索是怎麼實作的" --top-k 5
想完全離線、暫時不呼叫 Claude，把環境變數 ODM_EXTRACT_KNOWLEDGE 設為 false 即可關閉知識萃取，其餘流程照常運作。
三種介面

同一份資料，三種用法

命令列、MCP 伺服器與網頁主控台共用同一條管線、同一個資料庫。 不論你從哪裡匯入、又從哪裡查詢，看到的結果都一致。

⌨️

命令列

最直接的方式，四個指令就夠：ingest、search、stats、clear。

opendomainmcp stats
opendomainmcp clear
🔌

MCP 伺服器

透過 stdio 把工具提供給支援 MCP 的客戶端（例如 Claude）： ingest_path、search_knowledge、ask、 get_stats、list_collections。

opendomainmcp-server
🖥️

網頁主控台

啟動後在瀏覽器即可完成全部操作，適合一邊匯入、一邊觀察結果。

opendomainmcp-web
# http://127.0.0.1:8000
Dashboard

網頁主控台一覽

六個分頁，從檢視現況、匯入、搜尋、問答，到逐筆編輯與調整設定，都在同一處完成； 左側並可建立與切換多個知識庫。

📊

總覽 Dashboard

掌握資料庫的當前狀態：已索引的片段數量、使用中的嵌入模型與維度、資料存放位置，以及知識萃取是否啟用。 側欄的「知識庫」切換器可在不同專案之間切換或新建。

📥

匯入 Ingest

上傳檔案或填入路徑後，進度會即時回報；從載入、切塊、萃取、嵌入到入庫，每個階段都看得到。 勾選「Sync」還能在重新匯入時，順手清除已刪除檔案留下的舊片段。

🔎

搜尋 Explore

採 dense 與 BM25 混合檢索，對精確的符號名也找得準；可再以語言或來源路徑過濾。 每張卡片標示來源、符號與相似度，並附上該段摘要與關鍵概念。

💬

問答 Ask

把問題交給系統，它會先找出最相關的片段，再由 Claude 寫出一段附來源引用的答案（需設定 API 金鑰）。

✏️

瀏覽與編輯 Browse

以列表檢視所有已存內容，可逐筆查看、就地修正中繼資料，或刪除不再需要的項目。

⚙️

設定 Settings

嵌入後端、萃取模型、切塊大小、檢索模式等參數，都能在頁面上直接調整；變更會寫入設定檔並立即套用。

FAQ

常見問題

一定要付費的 API 金鑰嗎？

向量化（嵌入）與混合搜尋預設使用本地模型，不需任何金鑰。只有兩項會呼叫 Anthropic API： 知識萃取與「Ask」問答，兩者皆從 ANTHROPIC_API_KEY 與 ANTHROPIC_BASE_URL 取得設定。 不需萃取時把 ODM_EXTRACT_KNOWLEDGE 設為 false 即可；沒有金鑰時，Ask 會明確提示需要設定，而非回空答案。

可以同時管理多個專案的知識嗎？

可以。系統支援多個「知識庫」（collection），在側欄即可建立與切換，彼此資料互相隔離； 命令列用 --collection、MCP 工具用 collection 參數同樣可指定。

我的資料存放在哪裡？

預設在專案下的 .opendomain/ 目錄：向量資料庫位於 chroma/， 網頁上調整的設定則記在 settings.json。若想換位置，設定 ODM_DATA_DIR 即可。

重複匯入同一份檔案，內容會重複嗎？

不會。每個片段以「來源、行號、內容」三者的雜湊值作為識別碼；只要內容沒變，就視為同一筆， 重新匯入只是覆寫，不會新增。

遇到不支援或讀不了的檔案，會怎麼處理？

系統採「明確回報」的原則：略過的檔案與發生的錯誤都會逐一記錄並回報，不會默默吞掉； 同時，整體匯入仍會繼續處理其餘檔案，不致中斷。

可以改用品質更好的嵌入模型嗎？

可以。嵌入後端是可替換的，內建本地（fastembed）、OpenAI 與 Voyage 三種； 在設定頁或環境變數切換即可。細節請見技術文件的「嵌入層」一節。
